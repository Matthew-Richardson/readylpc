General

How the Site Works

ReadyLaPlata is a public-facing evacuation information system that combines:

  • ArcGIS Online maps displaying real-time evacuation zones
  • incident.json file containing incident details (shelters, check-in locations, road closures, animal services)
  • Automatic zone status pulled from the GIS layer attributes

When an incident is active, staff update the GIS layer zones and the incident.json file. The public site automatically reflects these changes.

Incident Naming Guide

Use the following format for all incident names:

[Name] + [Incident Type]

Rules:

  • Capitalize the first letter of each word
  • Incident types: Fire, Flood, Hazmat, Mudslide, etc.
Examples

Mesa Fire · Animas Flood · Highway 550 Hazmat

Zone Naming Guide

Use the following format for evacuation zone names:

[First 3 letters of incident] + "-" + [Unique #]

Rules:

  • Use ALL CAPS
  • Numbers must be unique across all active incidents
  • Keep numbering sequential within each incident
Examples

MES-1 · MES-2 · ANI-3 · HWY-4

Important

Zone names in the GIS layer must exactly match what appears in incident.json for proper linking.

Incident Form Fields Guide

The incident.json file contains all incident details displayed on the public site. Each field must follow the naming conventions above.

checkInAddr Check-in address Durango High School

CR 502
smallAnimals Small animal shelter name La Plata County Humane Society
smallAnimalsAddr Small animal shelter address 1111 S Camino Del Rio, Durango, CO 81303

largeAnimalsAddr Large animal shelter address 2500 Main Ave, Durango, CO 81301
Tip

Use line breaks within fields to display multiple items. Each line will appear on a separate row in the public interface.

Troubleshooting Guide

Common Issues

Zones not appearing on map

Check: Is the zone's status attribute set? Zones without a status ('Go', 'Set') won't display. Also verify the incident field matches the active incident name.

Incident info not showing

Check: Verify incident.json is valid JSON and the incident name exactly matches what's in the GIS layer's incident field.

Changes not appearing on public site

Check: GIS layer changes may take 1-2 minutes to propagate. For incident.json changes, ensure the file was uploaded to the IIS server and clear browser cache.

KML export failing

Check: Ensure you have zones with valid geometries selected. The export tool requires at least one zone to be checked.

Still having issues?

Contact GIS staff or check the browser developer console (F12) for error messages.

Deploying Updated JSON to Production

Export the JSON
Use the "Download JSON" button in the editor to save your incident.json file.
Navigate to the IIS folder
Open the web server directory:
C:\inetpub\wwwroot\readylpc
Delete the existing file
Remove the current incident.json from the folder.
Copy the new file
Paste your updated incident.json into the folder. Changes will be live immediately.
Important

Always verify changes on the live site after deploying. If the site shows errors, check that the JSON file is valid.
